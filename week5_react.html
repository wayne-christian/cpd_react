Main Ideas Behind React Components, Rendering & Lifecycles







Components

Return a node

Represent a piece of UI

Can be created in one of two ways:

1.Stateless (function)

2. Stateful (class)

Components

Stateless Components

Don’t need state or lifecycle methods

Any update in the props will trigger a rerender

A stateless component has no state(obvious, isn’t it?), it means that you can’t reach `this.state` inside it. It also has no lifecycle so you can’t use componentDidMount and other hooks.

Props vs State in React

In a React component, props are variables passed to it by its parent component. State on the other hand is still variables, but directly initialized and managed by the component. The state can be initialized by props.

    
<ChildComponent color=green />

  constructor(props) {
    super(props)
    console.log(props.color)
  }
}
